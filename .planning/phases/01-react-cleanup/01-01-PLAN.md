<objective>
Refactor `src/hooks/useMIDIInput.ts` so that the MIDI note-on handler and the
device-list-change listener attach exactly once on mount and read fresh state
through refs. This eliminates dependency churn (listeners re-attaching every
time config or connection state changes) and stale closures (handlers reading
mount-time values of config).

and MEM-03 (no stale closures in event handlers — handlers read fresh state via refs).

Output: A single modified file, `src/hooks/useMIDIInput.ts`, with two new refs
(`configRef`, `stateRef`), two cheap state-sync useEffects, and the two listener
useEffects rewritten with empty dependency arrays.
</objective>
