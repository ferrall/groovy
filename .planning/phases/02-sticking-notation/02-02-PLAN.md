  <name>Task 1: Extend ABCTranscoder.ts to render sticking annotations in notes view (per D-06)</name>
  <files>src/core/ABCTranscoder.ts</files>
  <action>
Modify the ABCTranscoder to include sticking annotations in the ABC notation output.

Understanding ABC annotations:
- ABC notation supports text annotations above notes: ^"text"
- Example: ^"R" above a note renders "R" above the staff
- Multiple annotations on same note are stacked

Sticking annotation rendering (per D-06):
- When a measure has sticking values, insert ABC annotations above the corresponding positions
- Only render non-null sticking values (empty/null values produce no annotation)
- Format: ^"L", ^"R", ^"L/R" (match the StickingValue strings exactly)
- Place annotations at the same position as the note/rest

Implementation approach:
1. Read the grooveToABC function signature and understand how it loops through measures and positions
2. Extend the position-level ABC generation (likely in generatePositionABC or similar) to include sticking annotations
3. Access measure.sticking array alongside measure.notes
4. For each position, if sticking exists and is non-null:
   - Append annotation to the ABC symbol: symbol + ^"stickingValue"
   - Example: "^g" (kick) becomes "^"R"^g" when sticking is R

Technical details:
- Sticking array must be read from measure.sticking (optional field, may not exist in older grooves)
- If sticking array doesn't exist or is empty, render no annotations
- Sticking length must equal notesPerMeasure (this is the component's invariant; ABCTranscoder trusts it's correct)
- Annotations are prepended to the ABC symbol so they appear above when rendered

Edge cases:
- Collision with tempo markings: D-06 says "move sticking slightly higher" — this is ABCjs rendering behavior (automatic, no code change needed)
- Multiple instruments on same beat: They share one sticking annotation (global per subdivision, per D-01)
- Rests: Annotations can appear above rests (valid ABC)

Backward compatibility:
- If measure.sticking is undefined or missing, skip annotation rendering (no error)
- Existing grooves without sticking render without annotations

Example:
Before (no sticking):
  ^g [c d] x | ...

After (with sticking: [R, L, R]):
  ^"R"^g ^"L"[c d] ^"R"x | ...

Find and update the measure action handlers (duplicateMeasure, deleteMeasure, clearMeasure) to correctly manage sticking data.

Behavior per D-09:
- **Duplicate measure:** Copy sticking along with notes
  - Code: When duplicating, copy both measure.notes and measure.sticking to the new measure
  - Implementation: setGroove(prev => ({ ...prev, measures: [...prev.measures.slice(0, index+1), duplicatedMeasure, ...prev.measures.slice(index+1)] }))
  - Duplicated sticking should be a copy of the array (not a reference) to prevent accidental linking

- **Delete measure:** Sticking data is deleted (automatic; it's part of the measure)
  - Code: Filter out the measure at the index
  - No special sticking handling needed; deletion is implicit

- **Clear measure:** Both notes AND sticking are reset to empty
  - Current behavior: Clears measure.notes (all false, or empty arrays)
  - New behavior: Also clear measure.sticking (set to null array or array of nulls)
  - Implementation: setGroove(prev => ({ ...prev, measures: prev.measures.map((m, i) => i === index ? { ...createEmptyMeasure(...), sticking: createEmptySticking(...) } : m) }))

Where to find these handlers:
- Search ProductionPage.tsx for "duplicateMeasure" or "addMeasure"
- Search for "clearMeasure" or "reset" for the clear handler
- Search for "deleteMeasure" or "remove" for the delete handler
- They may be in ProductionPage directly or called from useGrooveActions hook

Implementation specifics:
- Use createEmptyMeasure and createEmptySticking helpers (from types.ts)
- Ensure sticking array length = subdivision count for the measure
- Deep copy sticking on duplicate (use [...oldSticking] or Array.from() to avoid reference sharing)

- New Measure 2 has sticking: [R, L, R, L, R, L, R, L] (identical copy)
