Create a helper function findSimilarMeasures(groove: GrooveData, targetMeasureIndex: number): number[] that identifies measures with matching note patterns.

Similarity definition (per D-05):
- Same time signature
- Same subdivision count (notesPerMeasure)
- Same playable notes at same instrument positions (including articulation differences)

Note comparison:
- Notes are stored as Record<DrumVoice, boolean[]> in each measure
- Two measures are similar if their note structures are identical
- Articulation differences (hihat-open vs hihat-closed) count as different
- Exclude the target measure itself from results

Algorithm:
1. Get target measure's time signature and subdivision count
2. Get target measure's note positions and voices (flatten to set of "voice at position")
3. For each other measure:
   a. Check time signature matches
   b. Check subdivision count matches
   c. Compare note patterns:
      - For each voice, check if boolean arrays match element-by-element
      - If all voices match, measures are similar
4. Return array of indices of similar measures (excluding target)

Implementation specifics:
- Use a helper: areNotesIdentical(notes1, notes2): boolean that compares all voices
- For each voice, compare notes[voice] as arrays (true/false at each position)
- Example:
  ```
  function areNotesIdentical(notes1: Record<DrumVoice, boolean[]>, notes2: Record<DrumVoice, boolean[]>): boolean {
    return ALL_DRUM_VOICES.every(voice =>
      notes1[voice].length === notes2[voice].length &&
      notes1[voice].every((v, i) => v === notes2[voice][i])
    );
  }
  ```

Articulation handling:
- Since voices include articulation variants (hihat-open, hihat-closed are different voices in DrumVoice union)
- The note comparison already accounts for articulation (different voices = different patterns)
- No special articulation logic needed

Performance:
- For up to 16 measures, O(n * subdivisions) is acceptable
- No need for caching or optimization in Plan 3 scope

Example:
- Measure 1: Hi-hat on all 8ths, snare on 2&4 (sticking: [R, L, R, L, ...])
- Measure 2: Hi-hat on all 8ths, snare on 2&4 (no sticking)
- Measure 3: Hi-hat on all 16ths, snare on 2&4 (different subdivision)
- Measure 4: Hi-hat on all 8ths, snare on 1&3 (different notes)
- findSimilarMeasures(groove, 0) returns [1] (only Measure 2 matches)

Button logic:
1. User clicks "Apply to Similar Measures" button on measure N
2. Find similar measures using findSimilarMeasures(groove, N)
3. If similar measures found:
   a. Get current measure's sticking: groove.measures[N].sticking
   b. For each similar measure index M:
      - Copy sticking to groove.measures[M].sticking (deep copy)
   c. Update groove state with all changes
   d. Show success toast: "Applied sticking to {count} similar measures"
4. If no similar measures found:
   a. Show message to user: "No similar measures found."
   b. No groove state change

Handler implementation:
```typescript
const handleApplyToSimilar = (measureIndex: number) => {
  const similar = findSimilarMeasures(groove, measureIndex);
  if (similar.length === 0) {
    // Show toast: "No similar measures found."
    return;
  }
  
  const sourceSticking = groove.measures[measureIndex].sticking;
  if (!sourceSticking) {
    // No sticking to apply
    return;
  }
  
  setGroove(prev => ({
    ...prev,
    measures: prev.measures.map((m, i) =>
      similar.includes(i)
        ? { ...m, sticking: [...sourceSticking] } // Deep copy
        : m
    ),
  }));
  
  // Show toast: `Applied sticking to ${similar.length} similar measures`
};
```
