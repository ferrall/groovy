<objective>
Fix the residual performed-BPM display bias at the display level. Playing exactly on time at 120 BPM currently reads 120.5-121.5 because the EWMA averages per-interval BPM rates, which is upward-biased under human timing jitter (Jensen's inequality). The estimation math is being left as-is (a period-averaging alternative was tested and rejected for over-correcting dense playing). Instead, `getPerformedBpm()` snaps estimates within ¬±1 BPM of the set tempo to the set tempo, and rounds everything else to an integer ‚Äî because sub-¬±1 residual is noise, not signal.

Purpose: Make the "Playing:" readout trustworthy so drummers can act on it.
Output: Updated `getPerformedBpm()` + tests.
</objective>
